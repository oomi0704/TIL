## 1012 - Amazon Connect
- クラウドベースの**コンタクトセンター（コールセンター）サービス**
- 物理機器不要・従量課金
- 主な機能：受電・発信管理、IVR、通話ルーティング、チャット対応、通話録音・分析
- 連携：Amazon Lex（AIボット）、Lambda（カスタムロジック）、S3（録音保存）
- 試験キーワード：「コールセンターを構築したい」→ Amazon Connect

## 1012 - Amazon Pinpoint
- **マーケティング向けマルチチャネル通知・メッセージ配信サービス**
- 配信チャネル：メール、SMS、プッシュ通知、音声メッセージ
- ユーザーのセグメント分け・キャンペーン管理・配信結果分析が可能

### SNS との違い
| | Pinpoint | SNS |
|---|---|---|
| 用途 | マーケティング・顧客向け | システム間通知 |
| ターゲティング | ○ できる | × できない |
| 分析機能 | ○ あり | × なし |

## 1013 - AWS Organizations と OU

### 構造
```
Root（ルート）
└── OU（組織単位）
    ├── OU（入れ子も可能）
    │   ├── AWSアカウント
    │   └── AWSアカウント
    └── AWSアカウント
```

- **OU（Organizational Unit）**：アカウントをグループ化するフォルダ
- OUにポリシーを適用すると配下のアカウント全てに適用される

### SCP（サービスコントロールポリシー）
- OUやアカウントに適用する権限の制限ルール
- IAMより上位の制限（SCPで拒否されたらIAMで許可しても使えない）

## 1013 - AWS Config
- **検知はできる、ブロックはできない**
- リソースの設定変更を記録・監視、コンプライアンス違反を検知
- ブロックしたい場合は他サービスと組み合わせる

| 目的 | サービス |
|---|---|
| 検知して自動修復 | Config + Lambda |
| 操作を禁止 | SCP / IAM |
| 不正アクセスをブロック | WAF / Shield |

- **クロスリージョンレプリケーションは可能**（リードレプリカを使用）

| 種類 | リージョン | 用途 |

| リードレプリカ | 同一 or 別リージョン | 読み取り負荷分散・DR対策 |

### Aurora のクロスリージョン
| 方法 | クロスリージョン |

## 1017 - AWS DMS（Database Migration Service）

### フルロードとCDCの違い
| | フルロード | CDC |
|---|---|---|
| 内容 | 既存データを全件コピー | 変更分だけをリアルタイム同期 |
| タイミング | 移行開始時に1回 | 継続的に同期 |

### 使い分け基準
| シナリオ | 使うモード |
|---|---|
| 小規模・停止OKな移行 | フルロードのみ |
| 大規模・停止NGな移行 | フルロード + CDC |
| 移行後も継続同期したい | CDCのみ |

- **CDCのみはNG**：既存データがコピーされず不完全になる
- 必ず最初に**フルロードがセット**で必要

## 1019 - 移行関連サービスの整理

| サービス | 目的 |
|---|---|
| Application Discovery Service | サーバー情報の収集 |
| Migration Evaluator | コストの試算 |
| Migration Hub Strategy Recommendations | 移行戦略の推奨（7R） |
| Migration Hub | 移行の進捗管理 |
| DMS | 実際のDB移行作業 |

### 7R
| 戦略 | 内容 |
|---|---|
| Retire | 廃止する |
| Retain | そのまま残す |
| Rehost | そのままAWSへ（Lift & Shift） |
| Relocate | VMwareごと移行 |
| Repurchase | SaaSに乗り換え |
| Replatform | 少し改修して移行 |
| Refactor | クラウドネイティブに作り直し |

## 1022 - DeletionPolicy Retain
- CloudFormationのスタック削除時にリソースを残す設定

| ポリシー | 内容 |
|---|---|
| Delete | リソースも削除（デフォルト） |
| **Retain** | リソースを残す |
| Snapshot | スナップショットを取ってから削除 |

- 試験キーワード：「スタック削除後もリソースを残したい」→ **Retain**

## 1022 - STS AssumeRole

| シナリオ | 内容 |

| フェデレーション | 社外IDプロバイダーのユーザーにAWSアクセス付与 |

## 1023 - セキュリティグループ vs ネットワークACL

| | セキュリティグループ | ネットワークACL |
|---|---|---|
| 適用レベル | インスタンスレベル | サブネットレベル |
| 許可ルール | ✅ | ✅ |
| 拒否ルール | ❌ **できない** | ✅ **できる** |
| ステートフル | ✅ | ❌ |
| デフォルト | 全拒否 | 全許可 |

- 「特定IPを明示的に拒否したい」→ **ネットワークACL**

## CloudFront vs Global Accelerator（API用途）

| | CloudFront | Global Accelerator |
|---|---|---|
| キャッシュ | ✅ あり | ❌ なし |
| 動的コンテンツ・API | △ | ✅ 得意 |
| 静的コンテンツ | ✅ 得意 | △ |
| プロトコル | HTTP/HTTPS | HTTP・TCP・UDP |

- 「APIを高速化したい」→ **Global Accelerator**
- 「静的コンテンツを配信したい」→ **CloudFront**

## 1030 - AWS PrivateLink
- **VPC内からインターネットを経由せずプライベート接続できる仕組み**
- インターフェイス型VPCエンドポイント = PrivateLink

### VPCピアリングとの違い
| | VPCピアリング | PrivateLink |
|---|---|---|
| 接続範囲 | VPC全体 | 特定のサービスのみ |
| IPの重複 | ❌ 許可されない | ✅ 許可される |
| 方向 | 双方向 | 単方向 |

- 「IPが重複しているVPC間でサービス共有」→ **PrivateLink**

## BYOIP（Bring Your Own IP）
- **自社が所有するIPアドレスをAWSに持ち込んで使用できる機能**
- オンプレから移行してもIPが変わらない
- 使用できるサービス：EC2（Elastic IP）・Global Accelerator・NLB
- 要件：`/24`以上のIPブロック・自社所有の証明が必要
