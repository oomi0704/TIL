# AWS学習まとめ 1010-1019

## 1012 - Amazon Connect
- クラウドベースの**コンタクトセンター（コールセンター）サービス**
- 物理機器不要・従量課金
- 主な機能：受電・発信管理、IVR、通話ルーティング、チャット対応、通話録音・分析
- 連携：Amazon Lex（AIボット）、Lambda（カスタムロジック）、S3（録音保存）
- 試験キーワード：「コールセンターを構築したい」→ Amazon Connect

---

## 1012 - Amazon Pinpoint
- **マーケティング向けマルチチャネル通知・メッセージ配信サービス**
- 配信チャネル：メール、SMS、プッシュ通知、音声メッセージ
- ユーザーのセグメント分け・キャンペーン管理・配信結果分析が可能

### SNS との違い
| | Pinpoint | SNS |
|---|---|---|
| 用途 | マーケティング・顧客向け | システム間通知 |
| ターゲティング | ○ できる | × できない |
| 分析機能 | ○ あり | × なし |

- 試験キーワード：「マーケティングメール・SMSを一括配信」「ユーザーをセグメントして通知」→ Pinpoint

## 1013 - AWS Organizations と OU

### 構造
```
Root（ルート）
└── OU（組織単位）
    ├── OU（入れ子も可能）
    │   ├── AWSアカウント
    │   └── AWSアカウント
    └── AWSアカウント
```

- **OU（Organizational Unit）**：アカウントをグループ化するフォルダ
- OUにポリシーを適用すると配下のアカウント全てに適用される

### SCP（サービスコントロールポリシー）
- OUやアカウントに適用する権限の制限ルール
- IAMより上位の制限（SCPで拒否されたらIAMで許可しても使えない）
- 試験キーワード：「複数アカウント管理」→ Organizations、「権限を一括制限」→ SCP

## 1013 - AWS Config
- **検知はできる、ブロックはできない**
- リソースの設定変更を記録・監視、コンプライアンス違反を検知
- ブロックしたい場合は他サービスと組み合わせる

| 目的 | サービス |
|---|---|
| 検知して自動修復 | Config + Lambda |
| 操作を禁止 | SCP / IAM |
| 不正アクセスをブロック | WAF / Shield |

## 1014 - RDS クロスリージョンレプリケーション

- **クロスリージョンレプリケーションは可能**（リードレプリカを使用）
- マルチAZは同一リージョン内のみ

| 種類 | リージョン | 用途 |
|---|---|---|
| マルチAZ | 同一リージョン内 | 高可用性・自動フェイルオーバー |
| リードレプリカ | 同一 or 別リージョン | 読み取り負荷分散・DR対策 |

### Aurora のクロスリージョン
- **Aurora グローバルデータベース**：1秒未満のレプリケーション遅延、最大5リージョン（推奨）
- クロスリージョンリードレプリカも可能
- Auroraマルチマスターは同一リージョン内のみ（クロスリージョン不可）

| 方法 | クロスリージョン |
|---|---|
| Auroraマルチマスター | ❌ 同一リージョン内のみ |
| Auroraグローバルデータベース | ✅ 可能 |
| クロスリージョンリードレプリカ | ✅ 可能 |

## 1017 - Route53 CNAMEレコード
- **ドメイン名を別のドメイン名に紐付けるレコード**

### AレコードとCNAMEの違い
| | Aレコード | CNAMEレコード |
|---|---|---|
| 紐付け先 | IPアドレス | ドメイン名 |

### CNAMEの制約
- ゾーンの頂点（ルートドメイン）には使えない
- `example.com` には使えない ❌
- `www.example.com` には使える ✅
- ルートドメインに使いたい場合 → Route53独自の **Aliasレコード**

| | CNAME | Alias |
|---|---|---|
| ルートドメイン | ❌ 不可 | ✅ 可能 |
| AWSサービスへの紐付け | △ | ✅ 推奨 |
| 料金 | クエリ課金 | 無料 |

## 1017 - AWS DMS（Database Migration Service）

### フルロードとCDCの違い
| | フルロード | CDC |
|---|---|---|
| 内容 | 既存データを全件コピー | 変更分だけをリアルタイム同期 |
| タイミング | 移行開始時に1回 | 継続的に同期 |
| 対象 | 全データ | INSERT・UPDATE・DELETE |

### 移行フロー
```
① フルロード（既存の全データをコピー）
        ↓
② CDC開始（フルロード後の差分変更を継続同期）
        ↓
③ 切り替え（新DBに完全移行完了）
```

### 使い分け基準
| シナリオ | 使うモード |
|---|---|
| 小規模・停止OKな移行 | フルロードのみ |
| 大規模・停止NGな移行 | フルロード + CDC |
| 移行後も継続同期したい | CDCのみ |

### CDCのみだとどうなるか
- CDC開始以前の既存データはコピーされない
- 新DBのデータが不完全な状態になる
- 必ず最初にフルロードがセットで必要

## 1019 - AWS Application Discovery Service

- **オンプレミス環境のサーバー情報を自動収集・分析するサービス**
- 移行前の調査ツール

### 収集方法
| 方法 | 概要 | 環境 |
|---|---|---|
| Agent | サーバーにインストール | 物理・仮想問わず |
| Agentless Collector | 仮想アプライアンスをデプロイ | VMware環境のみ |

### AgentとCollectorの比較
| | Agent | Agentless Collector |
|---|---|---|
| インストール | 各サーバーに必要 | 不要 |
| 取得情報量 | 詳細 | 基本的なもののみ |
| ネットワーク依存関係 | ✅ 取得可能 | ❌ 取得不可 |

## 1019 - AWS Migration Evaluator

- **AWS移行のビジネスケース（費用対効果）を分析するサービス**
- 現在のオンプレ環境のコスト分析・AWS移行後のコスト試算
- 試験キーワード：「移行コストを試算したい」→ Migration Evaluator

---

## 1019 - AWS Migration Hub Strategy Recommendations

- **アプリケーションの移行戦略を自動で分析・推奨するサービス**
- 7Rのどれが適切か自動で提案してくれる

### 7R
| 戦略 | 内容 |
|---|---|
| Retire | 廃止する |
| Retain | そのまま残す |
| Rehost | そのままAWSへ（Lift & Shift） |
| Relocate | VMwareごと移行 |
| Repurchase | SaaSに乗り換え |
| Replatform | 少し改修して移行 |
| Refactor | クラウドネイティブに作り直し |

---
