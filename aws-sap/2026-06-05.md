# TIL - 2026年06月05日

## AWS SAP 学習メモ

---

### ASN（Autonomous System Number）
- インターネット上で自律システム（AS）を識別する番号
- BGPでルーティング情報を交換する際に使用
- AWS Direct ConnectやTransit GatewayでBGP設定時に必要

---

### ENI（Elastic Network Interface）
- VPC内の仮想ネットワークインターフェース
- ホットスワップ可能（実行中のインスタンス間で付け替え可能）
- セキュリティグループはENI単位で適用される
- MACアドレスに紐づくライセンス管理に活用可能

---

### Transit GatewayとオンプレミスのVPN接続
- VPN必須ではない
- **Site-to-Site VPN**：インターネット経由・低コスト
- **Direct Connect**：専用線経由・安定・低遅延
- Direct Connect + Transit Gatewayで複数VPCに一括接続可能

---

### CVE（Common Vulnerabilities and Exposures）
- 既知のセキュリティ脆弱性に付与される共通識別番号
- AWSでの対処サービス：
  - 脆弱性スキャン → **Amazon Inspector**
  - パッチ管理 → **Systems Manager Patch Manager**
  - 一元管理 → **Security Hub**

---

### Microsoft Defender vs Defender for Servers
- **Microsoft Defender**：Windows標準搭載のウイルス対策ソフト
- **Defender for Servers**：クラウド向け有料セキュリティサービス（EC2にも対応）
- 別製品・別カテゴリ

---

### ARN（Amazon Resource Name）
- AWSリソースを一意に識別する名前
- フォーマット：`arn:partition:service:region:account-id:resource`
- IAMポリシーやS3バケットポリシーでリソース指定に使用

---

### CORSポリシー
- 異なるオリジン間のリソース共有を制御する仕組み
- S3静的ウェブサイトやAPI Gatewayで設定が必要
- 「ブラウザからのアクセスがエラーになる」→ CORSを疑う
- サーバー間通信には関係なし

---

### CRMシステムとAWS連携
- SaaSとAWSの連携 → **Amazon AppFlow**
- コンタクトセンター構築 → **Amazon Connect**
- 顧客データ分析 → **Redshift + QuickSight**

---

### 機微性（きびせい）
- 情報の「扱いに注意が必要な度合い」
- 機微情報の検出 → **Amazon Macie**
- 機微情報の保存 → **Secrets Manager**
- 暗号化キー管理 → **KMS**

---

### CloudFront Functions vs Lambda@Edge

| | CloudFront Functions | Lambda@Edge |
|---|---|---|
| 実行時間 | 最大1ミリ秒 | 最大30秒 |
| 外部APIコール | 不可 | 可能 |
| コスト | 安い | 高め |
| 対応イベント | ビューワーのみ | 4つ全て |

---

### RAG（Retrieval-Augmented Generation）
- 外部の知識を検索してAIが回答を生成する仕組み
- AWS実装：**Knowledge Bases for Amazon Bedrock**
- ベクトル検索 → **Amazon OpenSearch**
- エンタープライズ検索 → **Amazon Kendra**

---

### Amazon Bedrock Guardrails
- AIの回答を安全・適切に制御する機能
- コンテンツフィルタリング・トピック拒否・機微情報マスキング・グラウンディング
- 入力・出力の両方をチェック

---

### 3大クラウド × AI

| クラウド | 主要AI | 提供サービス |
|---|---|---|
| AWS | Claude（Anthropic） | Amazon Bedrock |
| Azure | GPT-4o（OpenAI） | Azure OpenAI Service |
| Google Cloud | Gemini（自社） | Vertex AI |

---

### エニーキャストIP
- 同じIPアドレスを複数拠点に割り当て、最寄りにルーティング
- AWSでの活用 → **AWS Global Accelerator**
- CloudFrontとの違い：固定IP・TCP/UDP対応・キャッシュなし

---

### Route53
- VPC外のグローバルサービス
- プライベートホストゾーンでVPC内DNS解決も可能
- 加重ルーティング：トラフィックを割合で分散（Blue/Green・A/Bテストに活用）

---

### CloudFront + Auto Scaling
- 併用可能
- CloudFront（キャッシュ）→ ALB（負荷分散）→ Auto Scaling（EC2増減）
- CloudFrontのキャッシュによりスケールアウトを抑制できる

---

### Lambda SnapStart
- Javaランタイムのコールドスタートを最大90%削減
- デプロイ時に初期化済みスナップショットを作成
- 対応ランタイム：**Javaのみ**
