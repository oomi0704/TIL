## 2. 学んだこと（要点）

### クリックストリームデータ

- ユーザー操作を時系列で記録した行動ログ（ページ遷移、クリック、時刻など）。
- 分析用途は導線改善、離脱分析、レコメンド最適化。
- AWS では Kinesis -> S3 -> Athena/QuickSight の流れが典型。

### Athena と Glue

- Athena は S3 上のデータを SQL で直接分析するサービス（読む側）。
- Glue は Data Catalog/Crawler/ETL でデータを整理・変換するサービス（整える側）。
- 試験向けの覚え方は「Glue で整備、Athena で分析」。

### Storage Gateway

- 主目的はオンプレミスと AWS のハイブリッド接続。
- AWS 内（EC2）に配置して使う構成も可能。
- ただし一括移行の主役ではない。移行は DataSync/Snow Family/DMS が中心。

### ALB と NLB

- ALB: L7（HTTP/HTTPS）向け。パス/ホストベースルーティングが可能。
- NLB: L4（TCP/UDP/TLS）向け。高性能・低遅延・固定 IP が強み。
- 判定軸は「L7 の高度な振り分けが必要か」「UDP が必要か」。

## 3. 試験での使い分けメモ

- HTTP ヘッダやパスでルーティングしたい -> ALB
- UDP や超高スループットが必要 -> NLB
- S3 データをすぐ SQL で見たい -> Athena
- スキーマ管理や前処理まで含む -> Glue
- ハイブリッド接続を維持したい -> Storage Gateway
- 大量移行を短期間で実施したい -> DataSync/Snow Family/DMS
