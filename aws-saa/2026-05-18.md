## 1. ファイルシステム・ストレージサービスの使い分け

- **FSx for Windows** → SMBプロトコル、Windows環境のファイル共有
- **FSx for Lustre** → HPC・機械学習向け高速ストレージ
- **EFS** → LinuxのNFS共有、複数EC2から同時アクセス可能
- **Storage Gateway (File Gateway)** → オンプレのSMBをS3にマッピング
- **DataSync** → オンプレ ↔ AWSへの高速データ同期

> 💡 Windowsファイル共有 → FSx for Windows / Linux共有 → EFS

## 2. リージョン間プライベート接続

| 接続元 | 接続先 | サービス |
|---|---|---|
| オンプレ | AWS（インターネット経由） | Site-to-Site VPN |
| オンプレ | AWS（専用線） | Direct Connect |
| VPC | AWSサービス（S3など） | VPCエンドポイント |
| VPC 2つ | 別VPC（シンプル） | VPC Peering |
| VPC 多数・オンプレ混在 | 複数VPC | Transit Gateway |

- **VPC Peering** → 推移的ルーティング不可（A↔B、B↔CでもA→C不可）
- **Transit Gateway** → 推移的ルーティング可能、複雑な構成向け
- **Site-to-Site VPN** → オンプレ↔AWS専用、リージョン間VPC接続には不適切
- **VPCエンドポイント** → VPC→AWSサービスへの接続専用、VPC間接続には使えない

## 3. AWSの地理的概念

- **リージョン** → 地理的エリア（東京・バージニアなど）
- **AZ（アベイラビリティゾーン）** → リージョン内の複数データセンター群（物理的に離れている）
- **ローカルゾーン** → 特定都市に近い低レイテンシ拠点
- AZ間は低レイテンシの専用線で接続

## 4. メトリクス監視（CloudWatch）

- メトリクス監視 = CPU・メモリ・ネットワークなどの**数値データを監視**すること
- **CloudWatch** がAWSにおけるメトリクス監視の中心サービス
- デフォルトで**メモリ使用率は取得されない** → カスタムメトリクスで送信が必要
- EC2のデフォルト監視は**5分間隔**（詳細モニタリングで1分間隔に変更可・有料）

## 5. DAX vs ElastiCache Redis

| 項目 | DAX | ElastiCache Redis |
|---|---|---|
| 対応DB | DynamoDBのみ | RDS・DynamoDBなど何でも |
| 設定変更 | ほぼ不要 | アプリ側の実装が必要 |
| ユースケース | DynamoDB読み取り高速化 | セッション管理・ランキング |
| 永続化 | ❌ | ✅ |

**出題パターン**
- DynamoDB ＋ 最小限の変更 → **DAX**
- RDS ＋ キャッシュ → **Redis**
- セッション管理・ランキング → **Redis**
- シンプルなキャッシュ ＋ マルチスレッド → **Memcached**
