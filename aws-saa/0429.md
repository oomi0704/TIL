# 0429 AWS-SAAメモ（Amazon Polly関連）

## Amazon Polly の要点
- テキストを自然な音声に変換するTTSサービス
- 音声をリアルタイム配信、または音声ファイルとして保存可能
- 多言語対応、ニューラル音声対応

## Pollyに関連して出題されやすいサービス
- `Amazon Transcribe`
  - 音声 -> テキスト（STT）
  - コールセンターや会議ログの文字起こしでPollyと対比されやすい

- `Amazon Translate`
  - テキスト翻訳
  - 「Transcribeで文字起こし -> Translateで翻訳 -> Pollyで音声化」の構成で頻出

- `Amazon Comprehend`
  - 自然言語解析（感情分析、エンティティ抽出など）
  - チャット/問い合わせ分析とPolly連携のユースケースで登場

- `Amazon Lex`
  - 音声/テキストの対話ボット
  - Lex（会話制御） + Polly（音声応答）での組み合わせが定番

- `AWS Lambda`
  - サーバーレスでPolly呼び出し処理を実装
  - API Gateway連携の選択問題でよく出る

- `Amazon S3`
  - Pollyの音声ファイル保存先
  - ライフサイクル設定や配信（CloudFront）と合わせて問われる

- `Amazon Connect`
  - クラウドコンタクトセンター
  - 音声案内でPolly/Transcribeを使うシナリオが出題される

## 覚え方（試験向け）
- `Polly`: テキスト -> 音声
- `Transcribe`: 音声 -> テキスト
- `Translate`: テキスト -> 他言語テキスト
- `Lex`: 会話の制御
- `Comprehend`: 文章の意味解析

## 典型アーキテクチャ
- 多言語ボイスボット:
  - 音声入力 -> `Transcribe` -> `Translate` -> `Lex` -> `Polly` -> 音声出力
