## DR・データ移行・分析・運用（短い版＋試験キーワード）

| トピック | 一言 | 試験で引っかかるキーワード |
|----------|------|---------------------------|
| **AWS Elastic Disaster Recovery** | オンプレ／クラウドのサーバを **継続レプリ** し、障害時に **AWS へフェイルオーバー** する DR 用。 | RPO/RTO、エージェント、ステージング領域、訓練（ドリル）、CloudEndure 系の知識 |
| **DataSync ＋ Storage Gateway で VM/DB を「同期」** | **できない（用途が違う）**。DataSync は **ファイル・オブジェクト系のコピー**、Storage Gateway は **ハイブリッドストレージ**。VM 丸ごと・DB の本番移行／DR は別サービス。 | 誤答の取り合い、**MGN（アプリ移行）**、**Elastic DR**、**DMS（DB）**、スナップショット＋リストア |
| **クリックストリームデータ** | Web／アプリの **クリック・行動イベント** を連続で流すデータ。 | **Kinesis**（Streams / Firehose）、リアルタイム取り込み、分析パイプライン、イベントストリーム |
| **OpsCenter** | **Systems Manager（SSM）** の機能で、運用上の問題（OpsItem）を **一覧・追跡・対応** する。 | SSM、OpsItem、運用の一元化、インシデント／アラートと紐づけ（※「OpsWorks」と混同注意） |

## ハイブリッド：短い版＋試験の見分けキーワード

| サービス | 一言 | 試験で引っかかるキーワード |
|----------|------|---------------------------|
| **Storage Gateway** | オンプレと **S3 等ストレージ** をつなぐ（NFS/SMB/iSCSI）。 | ファイル共有、バックアップ、既存アプリのプロトコルそのまま、キャッシュ、テープ代替 |
| **Direct Connect** | **専用線**でオンプレ↔AWS。帯域・遅延が安定しやすい。 | 専用接続、大容量・長期転送、インターネットを避けたい、安定した帯域 |
| **Site-to-Site VPN** | **インターネット上**の IPsec トンネルでオンプレ↔VPC。 | すぐ繋ぐ、低コスト、暗号化VPN、DX のバックアップ経路 |
| **Transit Gateway** | **複数 VPC** とオンプレ（VPN/DX）を **ハブ** で集約。 | 多数VPC、ハブ＆スポーク、ピアリング整理、一元ルーティング |

**迷ったら**: ストレージ用途 → Gateway／線の質 → DX か VPN／VPCの数・設計 → TGW。
