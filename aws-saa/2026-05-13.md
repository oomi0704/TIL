# TIL - 2026-05-13 AWS SAA 学習メモ

---

## 1. リザーブドインスタンス vs Savings Plans

- **リザーブドインスタンス（RI）**：特定のインスタンスを予約。柔軟性低め。RDSはこちら。
- **Savings Plans**：一定の利用金額を約束。EC2・Lambda・Fargateに対応。AWSが推奨。
- Convertible RIはインスタンスタイプ変更可能だが割引率は下がる。

---

## 2. Transfer Family vs データ移行サービス

| サービス | 用途 |
|---|---|
| **Transfer Family** | SFTP/FTP等の既存プロトコルでS3/EFSに接続 |
| **DataSync** | オンプレ→AWSの定期自動同期 |
| **Snowball** | ネット困難・ペタバイト級の物理移行 |
| **DMS** | DB間の移行・レプリケーション |
| **Storage Gateway** | オンプレとAWSの常時ハイブリッド接続 |

> SFTPが出たら → Transfer Family

---

## 3. デプロイ戦略

| 戦略 | 内容 |
|---|---|
| **Canary** | 一部ユーザーだけ新バージョンに流す |
| **Blue/Green** | 新旧環境を丸ごと切り替え。ロールバック簡単 |
| **Rolling** | 少しずつ順番に更新 |
| **All at once** | 一気に全部更新。速いがリスク高い |

- Blue/Greenは環境が2つ必要でコスト高め
- Canaryは小さく試したい時

---

## 4. CloudFront vs Global Accelerator

| | CloudFront | Global Accelerator |
|---|---|---|
| 目的 | コンテンツキャッシュ配信 | 通信経路の最適化 |
| キャッシュ | あり | なし |
| 固定IP | なし | あり（2つ） |
| 用途 | 静的・動的コンテンツ | TCP/UDP・ゲーム・IoT |

## 5. EC2 スケーリング用語

| 用語 | 意味 |
|---|---|
| スケールアウト | インスタンスを**増やす** |
| スケールイン | インスタンスを**減らす** |
| スケールアップ | インスタンスを**大きくする** |
| スケールダウン | インスタンスを**小さくする** |

---

## 6. セキュリティサービス3つ

| サービス | 役割 |
|---|---|
| **GuardDuty** | 不正アクセス・異常行動の検知（警備員） |
| **Inspector** | EC2・Lambda等の脆弱性スキャン（検査員） |
| **Macie** | S3内の個人情報・機密データ検出（監査員） |

---

## 7. バックアップ・復旧

### PITRvsスナップショット

| | PITR | スナップショット |
|---|---|---|
| 復元粒度 | 秒単位で任意の時刻 | 取得時点のみ |
| 自動・手動 | 自動・継続的 | 手動orスケジュール |
| 主なサービス | DynamoDB・RDS・Aurora | EBS・RDS・Aurora |

### 災害復旧戦略（RTO/RPO順）

| 戦略 | コスト | 概要 |
|---|---|---|
| **Backup & Restore** | 安い | スナップショットから復元 |
| **Pilot Light** | 普通 | 最小限の環境を常時起動 |
| **Warm Standby** | 高い | 縮小版本番環境を常時起動 |
| **Multi-Site** | 最高 | 完全な本番環境を複数起動 |

- **RTO**：どれだけ早く復旧するか
- **RPO**：どこまで遡って復旧するか

## 8. EC2 ストレージ

| | インスタンスストア | EBS |
|---|---|---|
| 永続性 | ❌ 一時的 | ✅ 永続的 |
| 速度 | 爆速 | 普通 |
| EC2停止後 | データ消える | データ残る |
| 用途 | キャッシュ・一時ファイル | OS・DB・重要データ |

---

## 9. SPA（Single Page Application）

- HTML/CSS/JSで構成された1ページで動くWebアプリ
- 最初に1回だけHTMLを読み込み、以降はJSで画面を書き換える
- 例：Gmail・Google Maps・Twitter
- AWSでの典型構成：**S3（ホスティング）＋ CloudFront（配信）＋ API Gateway＋Lambda（バックエンド）**

---

## 10. AWS Amplify

- SPAなどフロントエンドを簡単にAWSへデプロイできるサービス
- S3・CloudFront・HTTPS・CI/CDを自動で構築
- 対応：React・Vue・Next.js・Angular
- 「素早く・簡単にフロントエンドをデプロイ」→ **Amplify**

---

## 本日の重要キーワードまとめ

> SFTP → Transfer Family  
> 固定IP・UDP → Global Accelerator  
> キャッシュ・コンテンツ配信 → CloudFront  
> 個人情報検出 → Macie  
> 脆弱性スキャン → Inspector  
> 不正アクセス検知 → GuardDuty  
> 簡単フロントデプロイ → Amplify
