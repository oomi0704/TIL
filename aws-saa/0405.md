# AWS SAA 向けまとめ（0405）

試験では **問題文のキーワード → サービス／設定** の対応を素早く取るのが近道です。下はこのファイルの用語を **SAA で使える形**に整理したものです。

---

## 1. ストレージ・アーカイブ

| 聞かれ方・キーワード | 選びやすい答えの型 |
|----------------------|-------------------|
| 長期保管・低コスト・取得に時間（分〜時間） | **S3 Glacier 系クラス**、**ライフサイクル**で段階的に移行 |
| 削除・変更をロック、コンプライアンス、WORM | **S3 Glacier の Vault Lock**、または文脈により **S3 Object Lock** |
| キーが偏ってスロットル、ホットパーティション | **キー設計の見直し**、**ランダムプレフィックス**（先頭を分散） |
| アクセスの少ないデータを安く | **ライフサイクル**で **Glacier** 等へ、**コールドデータ**という語 |
| オンプレを NFS/SMB/iSCSI のまま S3 とつなぐ | **Storage Gateway**（ファイル／ボリューム／テープ） |
| 「ホストする」＝置き場をクラウドへ | **S3 / EFS / FSx** など文脈に合わせて選択（ゲートウェイはハイブリッド橋） |

**誤答になりやすいもの**: アーカイブに **標準 S3 だけ**と書かれているのに高頻度アクセス前提、**DataSync** を「マウント共有の置き換え」と混同する。

---

## 2. 分析・DWH・データレイク

| 聞かれ方・キーワード | 選びやすい答えの型 |
|----------------------|-------------------|
| 大規模 OLAP、列指向 DWH | **Amazon Redshift** |
| Redshift から **S3 のデータを SQL** で読む | **Redshift Spectrum**（外部テーブル、レイク連携） |

**誤答になりやすいもの**: トランザクション OLTP を **Redshift** で、という文脈（なら **RDS／Aurora**）。リアルタイム集計だけなら **Athena** 単体の方が適する場合あり（問題文次第）。

---

## 3. DynamoDB・DAX・キャパシティ

| 聞かれ方・キーワード | 選びやすい答えの型 |
|----------------------|-------------------|
| 読み取りをマイクロ秒級に、キャッシュ | **DAX**（**書き込みは DynamoDB へ**、キャッシュは **最終的整合性**） |
| トラフィック予測できる、コスト安定 | **プロビジョンド**（**RCU/WCU**、**オートスケーリング**とセットで出る） |
| スパイクが読めない、キャパ計画したくない | **オンデマンド** |
| 特定パーティションに集中、スロットル | **ホットシャード**対策＝**均等なパーティションキー**、**ランダムプレフィックス** |

**誤答になりやすいもの**: 強い **直列整合性が必須の読み取り**を **DAX キャッシュだけ**に頼る。**Redis（ElastiCache）** と **DAX** の使い分け（DynamoDB 専用 API がテーマなら DAX）。

---

## 4. SQS（可視性タイムアウト）

| ポイント | SAA で覚えること |
|----------|------------------|
| 役割 | 受信後 **しばらく他コンシューマに見えない**秒数 |
| 短すぎる | 処理中に **再表示 → 重複処理** |
| 長すぎる | 障害時 **再処理が遅い** |
| 長時間ジョブ | **`ChangeMessageVisibility`** で延長、またはメッセージ分割など |
| 混同注意 | **ロングポーリング**（待ち方）は **別パラメータ** |

---

## 5. ネットワーク・接続のイメージ

| 用語 | SAA で一言 |
|------|------------|
| **エフェメラルポート** | クライアント側 **送信元の高位ポート**。SG／NACL の **戻り通信**説明で登場 |
| **踏み台（Bastion）** | **SSH 等の単一入口**でプライベートへ |
| **プロキシ** | **HTTP(S) 中継**、フィルタ・経路集約（SSH 専用入口とは別役割） |

## 7. 認証・IAM・API Gateway（典型パターン）

| シーン | SAA の正解の型 |
|--------|----------------|
| Lambda が DynamoDB / S3 を触る | Lambda の **実行ロール**に **最小権限ポリシー**（リソース ARN 指定）＋ログ用マネージドポリシー |
| API の **呼び出し元**を制限 | **IAM（`execute-api:Invoke`）**、**API リソースポリシー**、または **Cognito / Lambda オーソライザー** |
| Lambda の実行ロール | **バックエンド**用。**クライアント認可そのものではない**（問題で混ぜられる） |
| AWS 上のサービス主体 | **IAM ロール**（短期クレデンシャル） |
| 人・長期キーが必要なとき | **IAM ユーザー**（Lambda の実行には使わないのが基本） |
| 一時クレデンシャル | **STS**（`AssumeRole` 等） |
