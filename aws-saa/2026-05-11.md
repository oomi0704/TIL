## コンピューティング

### EC2 購入オプション
- **オンデマンド**: 短期・不定期な利用
- **リザーブド**: 1〜3年の長期利用でコスト削減
- **スポット**: 中断可能なバッチ処理など
- **Savings Plans**: 柔軟なコミットメント型割引

### Auto Scaling ポリシー
| ポリシー | 概要 |
|---|---|
| ターゲット追跡 | CPU使用率などを目標値に保つ |
| ステップ | 段階的にスケール |
| スケジュール | 時間ベースで事前にスケール |

### ELB の種類
| 種類 | 用途 |
|---|---|
| ALB (Application) | HTTP/HTTPS、パスベースルーティング |
| NLB (Network) | TCP/UDP、超低レイテンシ |
| GLB (Gateway) | サードパーティ仮想アプライアンス |

## ストレージ

### S3 ストレージクラス
- **Standard**: 頻繁なアクセス
- **IA / One Zone-IA**: 低頻度アクセス
- **Glacier / Glacier Deep Archive**: 長期アーカイブ
- **Intelligent-Tiering**: アクセスパターン不明時

### EBS / EFS / S3 の使い分け
| サービス | 用途 |
|---|---|
| EBS | EC2にアタッチするブロックストレージ |
| EFS | 複数EC2から共有できるファイルストレージ |
| S3 | オブジェクトストレージ（静的ファイル等）|

### EBS ボリュームタイプと IOPS
| タイプ | 特徴 |
|---|---|
| gp3 | 汎用SSD。IOPSを独立して設定可能 |
| io1 / io2 | 高IOPS必要なDBに使う（プロビジョンドIOPS）|
| st1 | スループット最適化HDD（ログ・動画向き）|
| sc1 | コールドHDD（アクセス頻度低い大容量）|

> 💡 **IOPSとは**: 1秒間にストレージが処理できる読み書きの回数  
> - **IOPS重視** → DBのトランザクション処理  
> - **スループット（MB/s）重視** → 大容量ファイルの転送

## データベース

### RDS Multi-AZ vs リードレプリカ
| 機能 | 目的 |
|---|---|
| Multi-AZ | **可用性**向上（フェイルオーバー用） |
| リードレプリカ | **読み取りパフォーマンス**向上 |

### Aurora の特徴
- ストレージが自動スケール
- グローバルDBで複数リージョン対応
- MySQL / PostgreSQL 互換

### DynamoDB
- **オンデマンド**: トラフィック予測不能な場合
- **プロビジョニング**: 安定したトラフィックでコスト削減
- **GSI**: 別のパーティションキーで検索可能
- **LSI**: 同じパーティションキー、別のソートキーで検索

### ElastiCache
| エンジン | 特徴 |
|---|---|
| Redis | 永続化・レプリケーション・複雑なデータ構造対応 |
| Memcached | シンプル・マルチスレッド・水平スケール |

## ネットワーク

### VPC 基本
- **IGW**: インターネットへの出入口
- **NAT Gateway**: プライベートサブネットからのアウトバウンド通信
- **ルートテーブル**: サブネット単位でトラフィックを制御

### セキュリティグループ vs NACL
| 項目 | セキュリティグループ | NACL |
|---|---|---|
| 適用対象 | ENI（インスタンス）レベル | サブネットレベル |
| デフォルト | 全拒否（インバウンド） | 全許可 |
| ルール | ステートフル | ステートレス |

### 接続方式
| サービス | 用途 |
|---|---|
| VPC Peering | VPC間の直接接続（推移的ルーティング不可）|
| Transit Gateway | 複数VPCのハブ接続 |
| Site-to-Site VPN | オンプレとの暗号化接続（インターネット経由）|
| Direct Connect | オンプレとの専用線接続（高帯域・低レイテンシ）|

## セキュリティ・IAM

### IAM 基本
- **ポリシー**: 権限の定義
- **ロール**: AWSサービスやユーザーに付与する権限セット
- **AssumeRole**: 一時的に別のロールの権限を取得

### 秘匿情報の管理
| サービス | 用途 |
|---|---|
| KMS | 暗号化キーの管理 |
| Secrets Manager | DBパスワード等の自動ローテーション対応 |
| SSM Parameter Store | 設定値・秘匿情報の保存（低コスト）|

---

## 疎結合・サーバーレス

### SQS / SNS / EventBridge の使い分け
| サービス | 用途 |
|---|---|
| SQS | キュー（1対1、順番処理）|
| SNS | Pub/Sub（1対多、即時配信）|
| EventBridge | イベントバス（AWSサービス連携・ルーティング）|

### Lambda
- イベント駆動のサーバーレス関数実行
- SQS・SNS・API Gatewayなどと組み合わせて使う

| サービス | 用途 |
|---|---|

## よく混同するペア まとめ

| ペア | 覚え方 |
|---|---|
| Multi-AZ vs リードレプリカ | 可用性 vs 読み取り性能 |
| Secrets Manager vs SSM | ローテーション必要 vs 設定値保存 |
| SQS vs SNS | キュー（1対1）vs Pub/Sub（1対多）|
| NAT Gateway vs NAT インスタンス | マネージド vs 自己管理（古い）|
| VPN vs Direct Connect | インターネット経由 vs 専用線 |
| IOPS vs スループット | 回数/秒 vs 転送量/秒 |
