# TIL - 2026-05-29 AWS SAA 勉強まとめ

---

## 1. SFTPアプリケーション
- SFTPはSSHベースの安全なファイル転送プロトコル（ポート22番）
- AWSでは **AWS Transfer Family** で実現
- バックエンドにS3やEFSを接続可能

---

## 2. Auto Scaling クールダウン
- スケーリング後に次のスケーリングを待機させる仕組み
- 過剰なスケーリングを防ぐ目的
- デフォルト300秒
- ステップ・ターゲット追跡スケーリングでは「ウォームアップ期間」を使う

---

## 3. Route53 フェイルオーバー
- **アクティブ・アクティブ** = 全エンドポイントが同時稼働・負荷分散（加重・レイテンシールーティング）
- **アクティブ・スタンバイ** = フェイルオーバーポリシーを使用・障害時に切り替え
- ヘルスチェックと組み合わせて自動切り替え

---

## 4. CloudHSM
- 専用ハードウェアで暗号鍵を管理するサービス
- AWSも鍵にアクセス不可（KMSとの最大の違い）
- FIPS 140-2 レベル3対応
- VPC内に配置
- オンプレHSMのクラウド版イメージ

---

## 5. ECS と ECR
- **ECR** = コンテナイメージの保管庫（DockerHubのAWS版）
- **ECS** = コンテナの実行環境
- ECSの起動タイプ：EC2タイプ（自己管理） vs Fargateタイプ（サーバーレス）

---

## 6. API Gateway プロキシ統合 / 非プロキシ統合
- **プロキシ統合** = リクエストをそのまま転送・シンプル・Lambda側で処理
- **非プロキシ統合** = マッピングテンプレート（VTL）でAPI Gateway側で変換・柔軟

---

## 7. Aurora グローバルデータベース vs レプリカ
- **レプリカ** = 同一リージョン内・読み取り分散・最大15台
- **グローバルDB** = 複数リージョン・災害対策・レプリケーション遅延1秒以下・RTO1分以内

---

## 8. Egress
- データが外に出ていく通信
- Ingressは入ってくる通信（基本無料）
- EgressはAWSで料金が発生することが多い
- **Egress-Only IGW** = IPv6専用のNATゲートウェイ的な役割・外向きのみ許可

---

## 9. SQSポリシー vs IAMポリシー
- 他のAWSアカウントへのアクセス許可は **SQSポリシー（リソースベースポリシー）** を使う
- IAMポリシーは自分のアカウント内のユーザー・ロールにしか設定できない

---

## 10. Global Accelerator
- TCP・UDP両方対応
- CloudFrontはHTTPのみ（UDPは非対応）
- 固定IPアドレスが2つ提供される
- CloudFront + NLBはHTTP/HTTPSのみ連携可能

---

## 11. AWS X-Ray
- アプリケーションの処理の流れを可視化・分析するトレーシングサービス
- セグメント・サブセグメントで処理を追跡
- EC2では **X-Rayデーモン** のインストールが必要
- CloudWatchはリソース監視、X-Rayは処理の流れ追跡

---

## 12. CloudWatch Logs
- AWSサービス・アプリのログを収集・保存・分析
- EC2は **CloudWatch Agent** のインストールが必要
- メトリクスフィルター・Logs Insightsで分析可能

---

## 13. CloudWatch Container Insights
- ECS・EKS専用のコンテナ監視機能
- コンテナ・タスク・サービス単位でCPU・メモリ等を監視
- **デフォルト無効** → 明示的に有効化が必要

---

## 14. EFS スループットモード
- **バースト** = クレジット制・不定期アクセス向き
- **プロビジョンド** = 固定速度・常時高負荷向き
- **エラスティック** = 自動調整・AWS推奨

---

## 15. AWS AIサービスまとめ

| サービス | 用途 |
|---|---|
| Comprehend | テキスト分析・感情分析・PII検出 |
| Textract | 画像・PDFから文字・表を抽出 |
| Transcribe | 音声 → テキスト |
| Polly | テキスト → 音声 |
| Lex | 会話AI・チャットボット |
| Forecast | 時系列の数値予測 |
| SageMaker | 本格的な機械学習基盤 |

---

## 16. SES vs SNS
- **SES** = メール専用・大量配信向き
- **SNS** = 複数チャネル（メール・SMS・Lambda・SQS）への通知
- 業務連絡・システム通知 → **SNS**
- メールマガジン・会員登録メール → **SES**

---

## 17. Windows/Linux ファイルストレージ
- **Windows** = FSx for Windows + SMBプロトコル
- **Linux** = EFS + NFSプロトコル
- FSx for Lustre = HPC・機械学習の高速ストレージ

---

## まとめ・気づき
- リソースベースポリシーはクロスアカウントアクセスに必須
- AIサービスは「何を入力して何を出力するか」で覚える
- WindowsはFSx、LinuxはEFSと対応を覚える
- 業務連絡系の通知はSNS（SESではない）
