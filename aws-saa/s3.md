# AWS SAA: S3 / Glacier 頻出まとめ

## ストレージクラス（頻出）

- **S3 Standard**
  - 高頻度アクセス向け
  - 低レイテンシで即時アクセス

- **S3 Standard-IA**
  - 低頻度アクセス向け（ただし即時取り出し可能）
  - ストレージ単価は安いが取り出し料金がかかる

- **S3 One Zone-IA**
  - 1AZ保存でさらに低コスト
  - AZ障害に弱いので再生成可能データ向け

- **S3 Intelligent-Tiering**
  - アクセス傾向が不明なときの定番
  - 自動で階層移動してコスト最適化

- **S3 Glacier Instant Retrieval**
  - アーカイブ用途だがミリ秒で取り出せる
  - 「普段読まないが、読む時は即時」が条件

- **S3 Glacier Flexible Retrieval（旧 Glacier）**
  - 取り出しは分〜時間
  - バックアップや中長期保管向け

- **S3 Glacier Deep Archive**
  - 最安クラス
  - 取り出しは長時間（12時間級）
  - 法定保存やほぼ読まないデータ向け

## 問題文での判断軸（この順で考える）

1. 取り出し速度は即時か、待てるか
2. アクセス頻度は高いか低いか
3. 保存コスト重視か、取り出し速度重視か
4. 複数AZ冗長性が必要か（One Zone可否）
5. 長期保存要件があるか

## 超頻出機能

- **Lifecycle**
  - 例: Standard -> Standard-IA -> Glacier へ自動移行
  - 期限で自動削除も可能

- **Versioning**
  - 削除ミス・上書きミス対策
  - MFA Delete が問われることがある

- **CRR (Cross-Region Replication)**
  - DR対策・コンプラ要件で頻出
  - 前提: バージョニング有効化

- **S3 Object Lock**
  - WORM（Write Once Read Many）で改ざん防止
  - 監査・規制対応で出題されやすい

## 典型パターン（即答用）

- 「最安で長期保存、ほぼ取り出さない」 -> **Glacier Deep Archive**
- 「アーカイブだが即時取り出し必要」 -> **Glacier Instant Retrieval**
- 「アクセスパターンが読めない」 -> **Intelligent-Tiering**
- 「低頻度だが即時アクセス必要」 -> **Standard-IA**
- 「1AZで問題なく、さらに安くしたい」 -> **One Zone-IA**
