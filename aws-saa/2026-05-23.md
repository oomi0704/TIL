## GWLB（Gateway Load Balancer）
- サードパーティのネットワーク仮想アプライアンス（ファイアウォール・IDS/IPSなど）をデプロイ・スケールするためのロードバランサー
- レイヤー3で動作し、GENEVEプロトコル（ポート6081）を使用
- 「サードパーティのファイアウォールを挟みたい」→ GWLB

## AWS Organizations と OU
- **Organization** = 会社全体のコンテナ（最上位）
- **OU（Organizational Unit）** = Organization内のグループ（フォルダ）
- SCPはRoot・OU・個別アカウントにアタッチ可能で、上から下へ継承される
- SCPはManagement Accountには効かない

---

## Control Tower
- マルチアカウント管理を自動化するツール
- 内部でOrganizationsを使用している
- Landing Zoneの自動構築・ガードレールの自動適用・Account Factoryによる自動プロビジョニングが可能
- 単純にアカウントをまとめたいだけなら Organizations で十分（Control Towerは過剰になる場合あり）

---

## KMS と Secrets Manager と Parameter Store

| サービス | 用途 |
|---|---|
| KMS | 暗号化キーの管理 |
| Secrets Manager | パスワード・APIキーの管理（自動ローテーションあり） |
| Parameter Store | 設定値・環境変数の管理（無料・ローテーションなし） |

- **自動ローテーションが必要** → Secrets Manager
- **コスト重視・設定値の管理** → Parameter Store
- Secrets ManagerはKMSと組み合わせて使うことが多い

---

## Elastic Beanstalk
- コードを渡すだけでインフラを自動構築・管理してくれるサービス
- サーバーレスではなく**マネージドサービス**（EC2が裏で動いている）
- Beanstalk自体は**無料**（裏のリソース代のみ課金）
- Dockerの知識不要・インフラ初心者向け

### 他サービスとの比較
```
EC2          → サーバーあり・自分で管理
Beanstalk    → サーバーあり・AWSが管理
ECS Fargate  → サーバーレス・コンテナ単位（Docker知識必要）
Lambda       → サーバーレス・関数単位
Lightsail    → AWS版レンタルサーバー・月額固定
```

## Storage Gateway ボリュームタイプ

### 保管型（Stored）
- **オンプレがメイン保管・AWSはバックアップ**
- 全データがローカルにあるので低レイテンシ
- オフライン環境でも動作可能
- 用途：災害対策・バックアップ

### キャッシュ型（Cached）
- **AWSがメイン保管・よく使うデータだけローカルにキャッシュ**
- 全データをS3に移してローカル容量を節約
- 用途：ストレージ容量の節約・クラウド移行

```
「災害対策・バックアップ」→ 保管型（Stored）
「容量節約・クラウド移行」→ キャッシュ型（Cached）
```

## ETL ジョブ
- **E**xtract（抽出）・**T**ransform（変換）・**L**oad（格納）
- データを取り出して・加工して・別の場所に格納する処理
- AWSでのETLサービス → **AWS Glue**
- データウェアハウス → **Redshift**
- S3のデータをSQLで分析 → **Athena**
