# TIL 2026-04-07（AWS SAA 関連）

## SAA 要点（試験で効くところだけ）

### 即効きするルール

- **明示的 Deny は最強** — どんな Allow より先に適用される。
- **S3 の Resource** — `...bucket/*` は**オブジェクト**だけ。`ListBucket` には **`...bucket`（`/*` なし）** も要る、という問題が出る。
- **IP 条件** — `aws:SourceIp` は**VPC エンドポイント経由**だと話が変わる／別条件が絡む、が落とし穴。
- **NotIpAddress + Deny** — 「指定 CIDR **以外**」から拒否、と読む（条件の真偽と Deny の組み合わせ）。

### サービスを選ぶとき（典型）

| キーワード | 答えになりやすいもの |
|------------|----------------------|
| VM **丸ごと**・リフト＆シフト・短いダウンタイムで移行 | **MGN**（Replication Agent） |
| **DB だけ**移行・継続複製 | **DMS**（＋RDS のレプリケーション系） |
| イメージを**一回**持ってくる寄り | **VM Import/Export**（継続追従なら MGN 寄り） |
| NAS から **EFS／S3 へ定期同期・大量転送** | **DataSync** |
| オンプレから **NFS／SMB で S3 を共有のように** | **File Gateway** |
| オンプレから **iSCSI ボリューム** | **Volume Gateway** |
| 既存が **テープバックアップ** | **Tape Gateway** |

### レプリケーション（試験の切り口）

- **同期** — 反映を待って成功扱いになりやすい／一貫性とトレードオフ。
- **非同期** — 速いがレプリカが一瞬古いことがある。
- **S3 CRR／SRR** — **バージョニング・ロール・暗号化**など**前提**とセットで出る。
- **RDS** — **Multi-AZ**＝高可用（待機系イメージ）、**リードレプリカ**＝**読み取り分散**／コピー。役割を混同しない。

### クリックストリーム

- **構成** — **Kinesis（Streams／Firehose）→ S3 → Athena**（など）のストリーム分析パターンを選べるようにする。

### 優先度低め（知っていれば十分）

- **Sid** — 主にラベル。試験の正解を直接決めにくい。

## IAM・S3 バケットポリシー

- **Sid（Statement ID）**  
  IAM／リソースポリシー内の各 Statement に付ける任意の識別子。運用・可読性用で、許可／拒否の判定ロジックには通常使われない。

- **Principal（プリンシパル）**  
  ポリシーが「誰に」効くかを指定する要素。バケットポリシーでは必須。`*` はすべての主体を対象にする。

- **NotIpAddress**  
  条件演算子の一つ。指定した IP／CIDR に**含まれない**とき条件が真になる。`Effect: Deny` と組み合わせると「その CIDR 以外からは拒否」という IP 制限が書ける。

- **明示的 Deny**  
  `Effect: Deny` のルール。他に Allow があっても**Deny が優先**される（SAA で頻出）。

- **aws:SourceIp**  
  リクエスト元 IP を条件に使うキー。**VPC エンドポイント経由**などでは条件の取り方が変わり、試験の落とし穴になりやすい。

- **S3 の Resource（バケットとオブジェクト）**  
  `arn:...:bucket/*` は**オブジェクト**のみ。`ListBucket` などには `arn:...:bucket`（末尾なし）も必要になることがある。

## 移行・VM レプリケーション

- **AWS Replication Agent**  
  ソース VM に入れる軽量エージェント。ディスクをブロック単位で送り、**増分**で追従。**MGN** の文脈で覚える。

- **Application Migration Service（MGN）**  
  サーバーを**リフト＆シフト**で AWS に移すサービス。レプリケーション・テスト起動・カットオーバーまで。対象は VM **丸ごと**。

- **増分レプリケーション（VM／ブロック）**  
  初回転送のあと**変わったブロックだけ**を送る方式。本番稼働中のまま切替準備ができ、ダウンタイムを抑えやすい。

- **AWS DMS（対比）**  
  **データベース**の移行・継続複製が主。VM 全体のディスクレプリケーションの代替にはならない。

- **VM Import/Export（対比）**  
  イメージのインポート等。**継続的な追従・短いダウンタイム**の移行なら **MGN** が答えになりやすい。

## ハイブリッドストレージ（DataSync と Storage Gateway）

- **AWS DataSync**  
  NFS／SMB 等と S3／EFS／FSx のあいだで、**大量データの移行・定期同期**を行うマネージド転送。スケジュール・帯域制御などが一体。

- **AWS Storage Gateway**  
  オンプレにゲートウェイを置き、NFS・SMB・iSCSI・仮想テープとして見せ、裏を **S3／FSx／Glacier** につなぐ。

- **File Gateway**  
  NFS／SMB でアクセス、実体は **S3**。

- **Volume Gateway**  
  **iSCSI ボリューム**として見せる。キャッシュ型／ストアド型など。

- **Tape Gateway**  
  バックアップアプリからは**仮想テープ**、実体は S3／Glacier 系。

| シナリオの目安 | 寄りがちな選択 |
|----------------|----------------|
| NAS から EFS へ定期同期・バッチ移行 | **DataSync** |
| オンプレから NFS／SMB で S3 を共有のように使う | **File Gateway** |

## レプリケーション（一般・SAA）

- **レプリケーション**  
  データや状態の**複製を別場所に持ち、更新を追従**させること。可用性・DR・読み取り分散・移行などが目的。

- **同期レプリケーション**  
  複製先への反映を待ってから成功とみなしやすい。一貫性は取りやすいが遅延・コストのトレードオフがありやすい。

- **非同期レプリケーション**  
  まずプライマリで成功し、あとから追従。速いが、短時間レプリカが古いことがある。

- **S3 CRR／SRR**  
  バケット間でオブジェクトを複製。**バージョニング・IAM ロール・暗号化**など前提とセットで出題されやすい。

- **RDS Multi-AZ とリードレプリカ**  
  Multi-AZ は高可用の主系／待機系イメージ。リードレプリカは**読み取り分散**や別 AZ／リージョンのコピー。役割の違いを問われやすい。

## 分析・ストリーミング

- **クリックストリーム（clickstream）**  
  Web／アプリ上の**クリック・ページビュー・遷移**などが時系列でつながったイベントデータ。分析・レコメンド・マーケに使う。

- **SAA での典型構成**  
  **Kinesis Data Streams／Firehose** で収集し、**S3** などに載せて **Athena** 等で分析するパターンがよく出る。
