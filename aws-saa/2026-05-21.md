# TIL - 2026年5月21日 AWS SAA 学習まとめ

---

## 1. ElastiCache for Redis
- RDS・DynamoDB・EC2アプリのキャッシュに使用
- Redisは**高可用性・永続化・マルチAZ対応**
- Memcachedはシンプルなスケールアウト向け

---

## 2. DynamoDB TTL（Time to Live）
- アイテムに有効期限を設定し**自動削除**する機能
- Unix時間（秒単位）で指定
- 削除は最大**48時間以内**に実行（即時ではない）
- TTL削除は**WCUを消費しない**
- 用途：セッション管理・OTP・キャッシュデータ

---

## 3. IAM と SCP の違い
- **IAM**：ユーザー・グループ・ロールへの権限管理（人単位）
- **SCP**：AWS Organizationsでアカウント・OU単位の制限
- SCPはあくまで**最大権限の境界**を設定するもの
- 実際の権限 = SCPとIAMの**積集合**

---

## 4. AWS Amplify
- フロントエンド・モバイルアプリを素早く構築するサービス
- 裏側：Cognito・AppSync・DynamoDB・S3・CloudFront
- 対応フレームワーク：React・Next.js・Vue・Flutter等
- GitHubと連携してCI/CDを自動化できる

---

## 5. エッジ関連サービスの整理

| サービス | エッジで何をするか |
|---|---|
| CloudFront | コンテンツを**キャッシュ**して配信 |
| Global Accelerator | エッジを**経由**してAWS内部網で高速化 |
| Route53 | エッジに**DNSサーバー**を配置 |
| S3 Transfer Acceleration | エッジ経由でS3に**高速アップロード** |
| WAF | エッジで**不正リクエストをブロック** |
| Shield | エッジで**DDoS攻撃を防御** |

### キーワード早見表
- キャッシュ・静的コンテンツ → **CloudFront**
- 固定IP・TCP/UDP・ゲーム・IoT → **Global Accelerator**
- S3アップロード高速化 → **S3 Transfer Acceleration**
- DNS・名前解決 → **Route53**

---

## 6. CloudFront vs Global Accelerator

| 項目 | CloudFront | Global Accelerator |
|---|---|---|
| キャッシュ | ✅ あり | ❌ なし |
| 固定IP | ❌ なし | ✅ 2つ発行 |
| 対象通信 | HTTP/HTTPS | TCP/UDP全般 |
| WAF適用 | ✅ 可能 | ❌ 不可 |

---

## 7. Route53 ルーティングポリシー
- **レイテンシーベース**：最も応答が速いリージョンへ転送
- **地理的ルーティング**：ユーザーの地域で固定振り分け
- **地理的近接性**：地理的に最も近いリソースへ転送
- **フェイルオーバー**：障害時に別リソースへ自動切り替え

---

## 8. データベース移行サービス

| サービス | 用途 |
|---|---|
| DMS | ダウンタイムなしのDB移行・リアルタイム同期 |
| SCT + DMS | 異なるDBエンジン間の移行 |
| Snowファミリー | 物理デバイスで大量データ移行 |
| DataSync | ファイルの定期同期（S3・EFS） |
| mysqldump + S3 | 小規模MySQLのシンプルな移行 |

---

## 9. AWS Snowball 使う条件
- データ量**10TB以上** かつ ネットワークが**遅い・不安定**
- 1Gbps回線で1週間以上かかる場合はSnowball検討
- 100PB以上 → **Snowmobile**

---

## 10. Amazon Redshift
- **データウェアハウス**サービス（分析・集計向け）
- 列指向ストレージ・並列処理（MPP）で大量データを高速分析
- RDSとの違い：日常業務→RDS、大量データ分析→Redshift
- **マルチAZ対応**（RA3ノードタイプ）
- Redshift Spectrum：S3のデータを直接クエリ

---

## 11. SQS
- メッセージ保持期間：デフォルト**4日**・最大**14日**
- メッセージサイズ：最大**256KB**
- インフライトメッセージ：標準**120,000件**・FIFO**20,000件**
- 標準キュー：順序保証なし・無制限スループット
- FIFOキュー：順序保証あり・重複排除

---

## 12. RDS マルチAZ の RPO・RTO
- **RPO**：ほぼゼロ（同期レプリケーションのため）
- **RTO**：1〜2分（自動フェイルオーバー）
- フェイルバック後の完全復旧もRTOに含む場合がある
- さらに高速な復旧が必要 → **Aurora**（30秒以下）

---

## 13. エラーページの切り替え

| シナリオ | 使うもの |
|---|---|
| 404・500エラーをカスタムページに | **CloudFront** |
| サーバーダウン時にメンテナンスページへ | **Route53フェイルオーバー** |

---

## 14. アプライアンス
- 特定機能に特化した専用機器・システム
- AWSでは**Gateway Load Balancer**でサードパーティアプライアンスを組み込める
- オンプレのファイアウォール代替 → **AWS Network Firewall**

```
キャッシュ → CloudFront
固定IP・TCP/UDP → Global Accelerator
S3アップロード高速化 → S3 Transfer Acceleration
ダウンタイムなし移行 → DMS
大量データ・オフライン移行 → Snowball
データ分析・ペタバイト → Redshift
順序保証・重複排除 → SQS FIFO
複数AZでファイル共有 → EFS

```
