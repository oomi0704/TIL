# TIL - 2026-05-24 AWS SAA 試験対策

## 本日学習したサービス

---

### 1. AWS Client VPN
- OpenVPNベースのリモートアクセスVPN
- 認証方式: Active Directory / 証明書 / SAML 2.0
- スプリットトンネル: AWS宛のみVPN経由（コスト削減）
- **Site-to-Site VPNとの違い**: Client VPNは個人ユーザー向け、Site-to-Site VPNは拠点間接続

---

### 2. AWS Directory Service
| 種類 | 特徴 |
|--|--|
| Managed Microsoft AD | フル機能AD・Trust関係可能・MFA対応 |
| AD Connector | オンプレADへのプロキシ・オンプレADが必須 |
| Simple AD | Sambaベース・小規模・MFA非対応 |

- **オンプレADをそのまま使いたい** → AD Connector
- **AWSとオンプレ両方** → Managed Microsoft AD + Trust

---

### 3. DynamoDB グローバルテーブル
- マルチリージョン・マルチアクティブ（全リージョンで読み書き可能）
- レプリケーションは自動・非同期（通常1秒以内）
- 競合解決: Last Writer Wins
- DynamoDB Streamsが内部で使用される

---

### 4. SSE-S3 vs SSE-KMS
| | SSE-S3 | SSE-KMS |
|--|--|--|
| 鍵管理 | AWSが完全管理 | KMSで管理 |
| 監査ログ | なし | CloudTrailで記録 |
| コスト | 無料 | KMS API呼び出し分課金 |

- **監査・コンプライアンスが必要** → SSE-KMS
- **シンプル・低コスト** → SSE-S3

---

### 5. AWS Outposts
- AWSのラックをオンプレミスに物理設置するサービス
- 利用可能: EC2・EBS・RDS・EKS・ECS・S3
- **ユースケース**: 低レイテンシ・データのオンプレ保存義務・ハイブリッドクラウド

---

### 6. AWS PrivateLink / VPCエンドポイント
```
VPCエンドポイント
├── ゲートウェイ型（S3・DynamoDBのみ・無料）
└── インターフェース型（その他AWSサービス・有料）← これがPrivateLink
```
- RDSはPrivateLinkに非対応（VPC内プライベートサブネットに配置する設計のため）
- 別会社へのDB共有: PrivateLink（リアルタイム）or KMS+スナップショット（コピー共有）

---

### 7. RDS 接続方法まとめ
| 状況 | 方法 |
|--|--|
| 同一VPC | セキュリティグループのみ |
| 別VPC（少） | VPCピアリング |
| 別VPC（多） | Transit Gateway |
| オンプレ（安定） | Direct Connect |
| オンプレ（安価） | Site-to-Site VPN |
| Lambda → RDS | RDS Proxy（接続数枯渇防止） |
| 認証情報管理 | Secrets Manager |

---

### 8. aws:NotIpAddress
- `Deny + NotIpAddress` = 指定IP以外を拒否 = **指定IPだけ許可**
- `Allow + IpAddress` とほぼ同じ動作だが、明示的Denyは他のAllowより優先されるため**セキュリティが強い**

---

### 9. S3ストレージクラスの使い分け
| クラス | アクセス頻度 | 最低保存期間 |
|--|--|--|
| Standard | 頻繁 | なし |
| Intelligent-Tiering | 予測不能 | なし |
| Standard-IA | 月1回程度 | 30日 |
| Glacier Instant Retrieval | 四半期に1回 | 90日 |
| Glacier Flexible | 年1回・数時間待てる | 90日 |
| Glacier Deep Archive | ほぼアクセスなし | 180日 |

---

### 10. Apache Parquet / Athena / S3 Select
- Parquet = 列指向フォーマット（分析・圧縮に最適）
- S3 Select: シンプルなフィルタ・単一ファイル・最安
- Athena: 複雑なSQL・不定期分析
- Redshift: 大規模・高頻度・常時分析

---

### 11. ACM（AWS Certificate Manager）
- SSL/TLS証明書を無料で発行・管理・自動更新
- 使えるサービス: CloudFront・ALB・API Gateway
- **EC2には直接使えない**
- **CloudFrontで使う場合はus-east-1で発行必須**

---

### 12. Amazon MSK（Managed Streaming for Apache Kafka）
- KafkaをAWSが管理するマネージドサービス
- Kafka互換のため既存Kafkaからの移行が簡単
- MSK Serverless: 完全サーバーレス・使った分だけ課金
- **Kafka互換が必要** → MSK　**AWSネイティブ** → Kinesis

---

### 13. AWS Config
- リソースの設定変更を記録・評価するサービス
- ルールによるコンプライアンスチェック
- **自動修復**: Config + Systems Manager Automation
- CloudTrailとの違い: CloudTrail=誰が操作したか、Config=設定がどう変わったか

---

### 14. CloudFormation StackSets
- 複数アカウント・複数リージョンに一括デプロイ
- セルフマネージド: IAMロールを手動設定
- サービスマネージド: Organizations連携・新規アカウントに自動デプロイ

---

### 15. AWS Service Catalog
- 承認済みAWSリソースのメニューをユーザーに提供
- ガバナンス・標準化・セルフサービスを実現
- CloudFormationテンプレートをベースに製品を登録

---

## 本日の重要ポイント

1. **権限管理の使い分け**
   - 過剰権限の検出 → IAM Access Analyzer
   - 設定変更の記録 → AWS Config
   - AWS全般のベストプラクティス → Trusted Advisor

2. **別会社へのDB共有**
   - リアルタイム接続 → PrivateLink（RDS前段にNLB必要）
   - データコピーを渡す → KMS + RDSスナップショット（KMSキーの共有が必須）

3. **Deny + NotIpAddress**
   - 指定IP以外を拒否 = 指定IPだけ許可
   - Allow + IpAddressより明示的Denyが強いためセキュリティが高い

4. **CloudFront + ACM**
   - 証明書は必ずus-east-1（バージニア北部）で発行

5. **Lambda → RDS**
   - 接続数枯渇を防ぐために RDS Proxy を使用
