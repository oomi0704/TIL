## SSL / TLS 認証（一般）

- いわゆる **SSL** は実質 **TLS**（通信の暗号化＋身元確認）。
- **サーバ認証（一般的）**: サーバ証明書を CA が署名。クライアントはチェーン・ホスト名・有効期限を検証。
- **相互TLS（mTLS）**: クライアント証明書も要求（API・社内向けなど）。
- **TLS とアプリ認証は別**: TLS は「正しいサーバ・盗み見しにくさ」。ID/パスワードはアプリ層。
- **自己署名**: 一般ブラウザは信頼しない（開発・閉域）。

## AWS での TLS / 証明書（SAA で効くポイント）

### ACM（AWS Certificate Manager）

- パブリック証明書の発行・**DNS 検証**・**自動更新**がしやすい。パブリック証明書の管理は無料（LB 等のリソース料金は別）。
- **インポート証明書**は自動更新は自分任せになりがち。

### ALB / NLB

- **HTTPS リスナー**に ACM 証明書をアタッチして TLS 終端。
- クライアント↔LB は TLS、LB↔ターゲットは HTTP の構成が多い（要件次第でターゲットまで HTTPS）。
- **SNI** で複数ドメイン・複数証明書を 1 LB に載せられるイメージ。

### CloudFront

- 独自ドメインの HTTPS 用証明書は **ACM** が典型。
- **CloudFront 用 ACM はリージョンが us-east-1（バージニア北部）必須**。ALB 用は **ALB と同じリージョン**の ACM。→ **試験で落とし穴になりやすい**。

### API Gateway / S3

- **API Gateway**: カスタムドメインに ACM、エッジ最適化 or リージョンエンドポイント。
- **S3**: オブジェクトは HTTPS 可。独自ドメイン＋HTTPS＋キャッシュは **CloudFront + ACM +（必要なら）OAC** のパターンが答案になりやすい。

## Elastic IP（EIP）

- **固定のパブリック IPv4**（リージョンスコープ）。アカウントで確保し **ENI に関連付け**。
- 通常の EC2 パブリック IP は停止・起動で変わり得るが、EIP は **付け替えつつ同じアドレス**を維持しやすい。
- **用途**: フェイルオーバーで IP を付け替え、相手先の IP ホワイトリスト用の固定出口など。
- **課金の方針**: 実行中インスタンスに 1 個付いている間は多く無料だが、**未使用の割り当て**や**停止中だけに付いたまま**などは課金されやすい → 不要なら **解放**。**NAT Gateway** は固定出口用に EIP を使う、とセットで覚えるとよい。
- **IPv6** は EIP と同じモデルではない（サブネット割り当てが中心）。

## Amazon AppFlow

- **SaaS ↔ AWS**（および一部外部）の **ノーコード寄りのデータ連携**（Salesforce、Slack、Zendesk 等 → **S3**、**Redshift**、Snowflake 等）。
- スケジュール・マッピング・簡易変換をコンソールで設定。
- **他サービスとの整理**: **Glue** は ETL・カタログ寄り、**DMS** は DB 移行/レプリケーション、**EventBridge** はイベントバス。問題文が「SaaS から S3 に定期取り込み」なら **AppFlow** が選択肢に乗りやすい。

## 1. 「言語・テキスト」に関する似ているサービス

文字や言葉を扱うサービス群です。

|サービス名|入力 (Input)|出力 (Output)|主な用途|
|---|---|---|---|
|Amazon Polly|テキスト|音声|ブログの読み上げ、自動アナウンス|
|Amazon Transcribe|音声|テキスト|会議の文字起こし、字幕作成|
|Amazon Translate|テキスト (言語A)|テキスト (言語B)|Webサイトの多言語化、翻訳|
|Amazon Comprehend|テキスト|感情・キーワード|顧客レビューが「喜び」か「不満」かの分析|

## 2. 「画像・動画」に関する似ているサービス

視覚情報を扱うサービスです。

|サービス名|役割|SAA試験でのキーワード|
|---|---|---|
|Amazon Rekognition|画像/動画分析|顔認証、不適切なコンテンツの検出、物体の特定|
|Amazon Textract|文字情報の抽出|紙の書類やPDFから「文字」や「表」をデータ化|

## SAA試験で特によく出る「ひっかけ」パターン

- 「文字起こし」をしたい場合
    
    - ❌ Polly (読み上げ用)
    - ⭕ Transcribe
    
- 「画像の文字（レシート等）」をデータ化したい場合
    
    - 🔺 Rekognition (物体認識は得意だが、文書の構造読み取りはTextract)
    - ⭕ Textract (OCRに特化)
    
- 「テキストの内容から顧客の怒り」を検知したい場合
    
    - ❌ Translate (翻訳するだけ)
    - ⭕ Comprehend (感情分析)
