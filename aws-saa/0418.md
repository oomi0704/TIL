# AWS SAA メモ（2026-04-18）

## CloudFront vs Global Accelerator（公式の整理）

- **CloudFront**: **HTTP / WebSocket** で配信できるコンテンツ向けの CDN。エッジキャッシュ・オリジンフェッチが中心。[Developer Guide / Introduction](https://docs.aws.amazon.com/AmazonCloudFront/latest/DeveloperGuide/Introduction.html)
- **Global Accelerator**: リスナーは **TCP・UDP・またはその両方**。静的 Anycast IP、AWS グローバルネットワーク経由の経路最適化・ヘルスチェック・フェイルオーバーが主役。**UDP 専用ではない**。[What is Global Accelerator?](https://docs.aws.amazon.com/global-accelerator/latest/dg/what-is-global-accelerator.html)
- **誤解しやすい点**: GA は **ALB（HTTPS 等）をエンドポイントにできる**ので Web 用途もある。切り分けは「**プロトコル HTTP だけ vs TCP/UDP**」より **キャッシュ CDN か／固定 IP・L4・マルチリージョン経路か**で見る方が試験でも実務でも合いやすい。
