# TIL - 2026-05-28 AWS SAA 学習メモ

## シングルサインオン関連サービス

### IAM Identity Center（旧AWS SSO）
- 複数のAWSアカウントやビジネスアプリに一度のログインでアクセス
- SAML2.0対応・AWS Organizationsと統合
- 権限セットで一元管理

### Amazon Cognito
- **User Pools**: アプリのサインイン・サインアップ管理→JWTトークン発行
- **Identity Pools**: ソーシャルログインでAWSリソースにアクセス→一時的IAM認証情報を発行
- 「モバイルアプリ認証・GoogleログインでS3アクセス」→ Cognito

### AWS Directory Service
- **AD Connector**: オンプレADへのプロキシ・AWSにADを作らない・オンプレ主体
- **AWS Managed Microsoft AD**: AWSにフルマネージドAD構築・オンプレADと信頼関係・AWS主体
- **Simple AD**: 小規模・低コスト・オンプレAD連携不可
- オンプレ主体（ADそのまま）→ AD Connector
- AWS主体（AWSにAD構築）→ Managed Microsoft AD

### AWS Organizations
- 複数AWSアカウントを一元管理
- OU（組織単位）でアカウントをグループ化
- 一括請求（Consolidated Billing）で使用量合算割引
- SCP（Service Control Policy）で権限の上限を設定
- Organizations + IAM Identity Center → 複数アカウントへのSSO

## Redis 関連

### ElastiCache for Redis vs MemoryDB for Redis
- **ElastiCache for Redis**: キャッシュ用途・低コスト・データ損失あり得る
- **MemoryDB for Redis**: 永続化対応・プライマリDB用途・ベクトル検索対応・高コスト
- 「キャッシュ」→ ElastiCache、「永続化・DB代替」→ MemoryDB

### ElastiCache Redis vs Memcached
- **Redis**: 複雑なデータ構造・レプリケーション・セッション管理向き
- **Memcached**: シンプルなキャッシュ・マルチスレッド

### Redisが連携しにくいサービス
- **S3** → CloudFrontが最適
- **DynamoDB** → DAXが最適
- **Redshift** → Redshift結果キャッシュ機能が最適
- **Athena** → クエリ結果再利用機能が最適
- **SQS・SNS** → そもそも用途が違う

### DAX（DynamoDB Accelerator）
- DynamoDB専用インメモリキャッシュ
- S3・RDS・Auroraとは連携不可
- DynamoDB APIと完全互換・アプリ変更がほぼ不要
- 「DynamoDB高速化」→ DAX一択

## RDS マルチAZ DBクラスター

### 通常のマルチAZとの違い
- **通常のマルチAZ**: 2インスタンス（プライマリ+スタンバイ）・スタンバイ読み取り不可・フェイルオーバー約60秒
- **マルチAZ DBクラスター**: 3インスタンス（1書き込み+2読み取り）・読み取り分散可能・フェイルオーバー約35秒

### リーダーエンドポイントが使えるもの
- Aurora ✅
- RDS マルチAZ DBクラスター ✅
- RDS 通常のマルチAZ ❌
- RDS シングルAZ ❌

### クロスリージョンでリーダーエンドポイントが必要な場合
- Auroraグローバルデータベースを使用
- フェイルオーバー1分以内・レプリケーション遅延1秒以下

## ワイルドカード証明書

### 概要
- `*.example.com` のようにサブドメインを一括カバーするSSL証明書
- 2階層のサブドメイン（www.sub.example.com）は対象外

### ACM（AWS Certificate Manager）
- ワイルドカード証明書を無料発行・自動更新
- 適用可能：ALB・CloudFront・API Gateway
- EC2には直接使用不可（サードパーティ証明書を使用）
- **CloudFrontで使う場合はus-east-1で発行が必須**

## DMS（Database Migration Service）の対応範囲

### 全方向の移行に対応
- オンプレ → AWS
- 他クラウド（Azure・GCP）→ AWS
- AWS → AWS（クロスリージョン・クロスアカウント）
- AWS → オンプレ

### 移行先として対応しているサービス
- RDS・Aurora・DynamoDB・S3・Redshift・OpenSearch・Kafka

### 異種DB移行
- SCT（Schema Conversion Tool）でスキーマ変換 + DMSでデータ移行をセットで使用

## 全体の使い分けまとめ

```
DB高速化：
　DynamoDB → DAX
　RDS・Aurora → ElastiCache Redis
　Redshift → 結果キャッシュ機能

コンテンツ高速化：
　S3・静的コンテンツ → CloudFront

SSO：
　複数AWSアカウント → IAM Identity Center
　アプリ認証 → Cognito
　AD連携 → Directory Service

DB移行：
　異種DB・停止できない → DMS
　同種DB・小規模 → mysqldump・スナップショット
```
